# PRD: продуктовые границы агентной системы

Этот PRD описывает текущую реализованную систему как основу для будущего продукта: что уже закрыто архитектурно, какие сценарии доступны пользователю и какие возможности остаются в зоне развития.

## Назначение продукта

Платформа должна дать архитектурной функции управляемый способ отвечать на вопросы по корпоративной базе знаний. Ответ должен быть не только “похож на правду”, но и иметь проверяемую связь с источниками.

## Основные пользователи

| Роль | Сценарий |
|------|----------|
| Руководитель архитектурной функции | Получить быстрый обзор по архитектурному вопросу и понять основание ответа. |
| Enterprise architect | Проверить политику, стандарт, техрадар или архитектурный принцип. |
| Оператор базы знаний | Загрузить документы и запустить ingestion. |
| Инженер / ML-команда | Развивать runtime, retrieval, ingestion и tools. |
| Эксплуатация | Контролировать готовность и деградацию зависимостей. |

## Реализованный MVP

| Capability | Статус | Значение |
|------------|--------|----------|
| Диалоговый агент | Реализовано | Единый контур ответа вместо набора разрозненных ролей. |
| RAG on-demand | Реализовано | Поиск вызывается через `search_kb`, когда агенту нужен контекст. |
| Citations | Реализовано | Ответ можно связать с найденными источниками. |
| Ingestion | Реализовано | Документы включаются в базу знаний через управляемый pipeline. |
| Async jobs | Реализовано | Загрузка знаний не блокирует пользовательский контур. |
| Health | Реализовано | Система различает “жива” и “готова отвечать”. |

## Не входит в текущий MVP

- Workspaces и изоляция команд.
- Аутентификация, роли, приглашения.
- Полноценный React-интерфейс.
- Drag-and-drop загрузка документов.
- Confluence, SQL/CMDB и web-коннекторы.
- MCP, агентные навыки и визуальные flows.
- Embed-widget для корпоративного портала.

## Продуктовая логика развития

Следующий этап должен сместить фокус с backend-ready платформы к self-service продукту для архитектурной функции. Приоритет: workspaces, загрузка документов, нормальный chat UI и RBAC.
