# Глоссарий архитектуры агентной системы

Глоссарий фиксирует язык, на котором удобно обсуждать платформу с C-level, архитекторами и инженерной командой. Английские термины оставлены там, где они являются отраслевым стандартом.

| Термин | Объяснение |
|--------|------------|
| Агент корпоративной архитектуры | LLM-контур, который отвечает на вопросы об архитектуре, стандартах, принципах и документах, используя инструменты и базу знаний. |
| Архитектурный контур | Совокупность интерфейса, агента, поиска, памяти, ingestion и эксплуатации, которая обеспечивает управляемый ответ на архитектурный вопрос. |
| Корпоративная база знаний | Набор документов, фрагментов и метаданных, подготовленных для поиска и использования агентом. |
| RAG | Retrieval-Augmented Generation: подход, при котором модель дополняет ответ найденными фрагментами из базы знаний. |
| RAG on-demand | Режим, где поиск вызывается только по решению агента, а не перед каждым ответом. |
| Tool loop | Цикл, в котором LLM может запросить инструмент, получить результат и продолжить рассуждение. |
| `search_kb` | Инструмент агента для поиска по корпоративной базе знаний. |
| Embedding | Числовое представление текста, позволяющее искать смысловую близость, а не только совпадение слов. |
| Vector store | Хранилище embeddings и метаданных; в архитектуре используется как контур семантического поиска. |
| Citation | Ссылка на источник или фрагмент, на который опирается ответ. |
| Трассируемость ответа | Способность показать, какие источники и tool calls повлияли на результат. |
| Session memory | Краткосрочный контекст текущего диалога; помогает сохранить нить разговора, но не является источником истины. |
| Ingestion | Процесс включения документов в базу знаний: загрузка, разбор, нормализация, chunking, embeddings, индексирование. |
| Chunk | Фрагмент документа, который индексируется и может быть найден retrieval-контуром. |
| Worker queue | Очередь заданий для управляемой обработки документов без блокировки пользовательского интерфейса. |
| Readiness | Готовность системы отвечать пользователю с учетом зависимостей: embeddings, vector store, storage, база данных. |
| Liveness | Факт, что приложение как процесс живо и отвечает на базовый health-запрос. |
| Workspace | Будущая продуктовая единица изоляции знаний, пользователей и настроек. В текущей архитектуре близка к `corpus_id`. |
| Санитизация | Подготовка документации к распространению без секретов, внутренних путей, приватных промптов и чувствительных деталей. |

## Как использовать этот словарь

- В executive-разделах опирайтесь на термины “архитектурный контур”, “корпоративная база знаний”, “трассируемость ответа”.
- В инженерных разделах используйте технические термины `RAG`, `tool loop`, `embedding`, `worker queue`.
- В продуктовых обсуждениях связывайте `workspace`, роли и document management с будущей Phase 2.
